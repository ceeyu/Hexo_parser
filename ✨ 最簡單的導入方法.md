### 使用方法（每次導入文章）

#### 步驟 1：在 iThome 打開文章

前往你的文章列表：
```
https://ithelp.ithome.com.tw/users/20151593/articles
```

點擊任一篇文章

#### 步驟 2：點擊書籤

點擊剛才創建的 `📝 複製為Markdown` 書籤

會彈出提示：`✅ 已複製 Markdown 格式！`

#### 步驟 3：創建 Hexo 文章

打開命令提示符，運行：
```cmd
node tools/快速創建文章.js
```

按提示輸入（或直接按 Enter 跳過，因為我們已經有內容了）

#### 步驟 4：貼上內容

1. 打開生成的 `.md` 文件
2. **全選並刪除**原有內容（Ctrl+A, Delete）
3. **貼上**剪貼板內容（Ctrl+V）
4. 保存

#### 步驟 5：預覽

```cmd
npm run server
```

訪問 http://localhost:4000 查看效果

#### 步驟 6：部署

```cmd
npm run clean
npm run build
npm run deploy
```

## 🔄 更簡單的方法：直接手動複製

如果書籤工具不work，用這個最原始但最可靠的方法：

### 步驟：

1. **打開 iThome 文章**

2. **創建 Hexo 文章**
   ```cmd
   node tools/快速創建文章.js
   ```
   輸入標題、標籤等

3. **複製 iThome 內容**
   - 在 iThome 文章頁面
   - 選取文章內容（從第一段開始）
   - Ctrl+C 複製

4. **貼到 Hexo 文章**
   - 打開生成的 `.md` 文件
   - 找到 `## 簡介` 下方
   - Ctrl+V 貼上
   - 保存

5. **調整格式**（如果需要）
   - 代碼塊加上 \`\`\`
   - 標題加上 ##
   - 列表加上 -

6. **預覽和部署**
   ```cmd
   npm run server
   npm run clean && npm run build && npm run deploy
   ```

## 🎨 格式調整技巧

### 代碼塊

如果代碼沒有語法高亮：

```markdown
# 改前
code here

# 改後
\`\`\`python
code here
\`\`\`
```

### 圖片

如果圖片是 iThome 的連結，可能會失效。建議：

1. 下載圖片到 `source/images/`
2. 修改連結：
```markdown
![圖片說明](/images/圖片名稱.jpg)
```

### 標題層級

確保標題層級正確：
```markdown
# 一級標題（文章標題，通常不用）
## 二級標題（主要章節）
### 三級標題（子章節）
```

## 🆘 常見問題

### Q: 書籤工具沒反應？
A: 
1. 確認完整複製了 javascript: 開頭的代碼
2. 確認在 iThome 文章頁面使用
3. 改用手動複製方法

### Q: 複製的格式很亂？
A: 
1. 手動調整 Markdown 格式
2. 重點檢查代碼塊和圖片
3. 參考其他正確的文章格式

### Q: 圖片無法顯示？
A:
1. 檢查圖片連結是否有效
2. 下載圖片到本地
3. 使用相對路徑

### Q: 一次導入太多文章會怎樣？
A:
1. 不會有問題，但建議分批
2. 每次部署前先本地預覽
3. 避免一次修改太多導致難以排查問題
